# Data Agent

A multi-agent data analysis system for querying Azure Cosmos DB logistics data using natural language. Built on [LangGraph](https://github.com/langchain-ai/langgraph) with an Azure OpenAI LLM backend.

## What it does

Given a natural-language question, the system:
1. **Plans** a step-by-step execution strategy
2. **Routes** each step to a specialized agent (order data, route data, or synthesizer)
3. **Returns** a prose answer, optionally with a matplotlib chart

**Example queries:**
- "How many routes were planned on April 10, 2026?"
- "Give me the number of routes for request id: cd847bd2-..."
- "What is the average vehicle weight utilization across all routes last week? Visualize with a bar chart."

## Setup

### Prerequisites

- Python 3.10+
- Access to Azure Cosmos DB (databases `hgs-input` and `hgs-output`)
- Azure OpenAI endpoint with `DeepSeek-V4-Flash` deployed

### Install dependencies

```bash
pip install langgraph langchain langchain-core langchain-openai langchain-community \
            langchain-tavily azure-cosmos python-dotenv pydantic matplotlib numpy llmclean
```

For notebook exploration only (not required for the main pipeline):
```bash
pip install snowflake-snowpark-python trulens langchain-experimental
```

### Configure credentials

Create a `.env` file in the project root:

```env
azure_openai_key=<your Azure OpenAI key>
cosmos_url=<your Cosmos DB endpoint>
cosmos_key=<your Cosmos DB key>
tavily_key=<your Tavily search key>
```

## Usage

Edit the `query` variable near the bottom of `data_agent.py`, then run:

```bash
python3 data_agent.py
```

Output: final prose answer printed to stdout, token usage summary, and `chart_output.png` saved to the project directory if a chart was requested.

To restrict which agents are used, set `enabled_agents` in the state:

```python
state = {
    "messages": [HumanMessage(content=query)],
    "user_query": query,
    "enabled_agents": ["cosmos_route"],  # only query route output data
}
```

## Architecture

```
User query
    │
    ▼
┌─────────┐     JSON plan      ┌──────────┐
│ planner │ ─────────────────► │ executor │
└─────────┘                    └──────────┘
                                     │
               ┌─────────────────────┼─────────────────────┐
               ▼                     ▼                       ▼
        ┌─────────────┐    ┌──────────────────┐    ┌─────────────┐
        │ cosmos_order│    │  cosmos_route    │    │ synthesizer │
        │ (hgs-input) │    │  (hgs-output)   │    │             │
        └─────────────┘    └──────────────────┘    └─────────────┘
```

| Agent | Data source | Purpose |
|---|---|---|
| `cosmos_order` | `hgs-input` / `orders` | Optimizer input: orders, customers, equipment available |
| `cosmos_route` | `hgs-output` / `route` | Optimizer output: planned routes, utilization, distances |
| `synthesizer` | Previous agent messages | Produces the final prose answer |

The planner and executor are LLM-driven. All routing after `planner` is dynamic — the executor decides which agent runs next and can trigger replanning (up to 3 attempts per step) if a step is blocked.

Charts are generated by the Cosmos agents via an LLM-written Python script executed as a subprocess. The PNG is base64-encoded into the graph state and saved to `chart_output.png` at the end of the run.

## Project structure

```
data_agent.py       # Graph definition, node implementations, __main__ entry point
cosmos_agent.py     # CosmosRouteAgent and CosmosOrderAgent (ReAct agents over Cosmos DB)
prompts.py          # Prompt builders and agent registry
helper.py           # python_repl_tool (chart execution), TruLens setup
search_agent.ipynb  # Interactive exploration notebook
cosmos_test.ipynb   # Cosmos DB query testing notebook
```

## Known quirks

- `MAX_REPLANS` is `3` in `data_agent.py` but the prompt text in `prompts.py` says `2`. Effective limit is `3`.
- `helper.py` imports Snowflake and TruLens at module load — if those packages aren't installed, the import will fail even though they aren't used by the main pipeline.
- `temp_agent_script.py` is machine-generated on each run; do not edit it manually.
